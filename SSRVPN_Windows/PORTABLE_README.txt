SSRVPN Windows 便携版 v2.0.1

═══════════════════════════════════════
  使用方法
═══════════════════════════════════════

1. 完整解压 ZIP 到文件夹（⚠️ 不要在压缩包预览窗口中直接运行）
2. 右键 ssrvpn_windows.exe → 属性 → 如果看到"解除锁定"，勾选后点确定
   这一步很重要！否则 Windows SmartScreen 可能静默拦截程序。
3. 双击 ssrvpn_windows.exe 启动
4. 粘贴你的订阅链接或节点配置
5. 打开主页的连接开关

═══════════════════════════════════════
  打不开怎么办？
═══════════════════════════════════════

第一步：双击运行 SSRVPN_Diag.bat
  → 自动诊断缺失文件、SmartScreen 拦截等问题
  → 生成详细诊断报告

第二步：如果诊断未解决，尝试安全模式
  → 双击运行 ssrvpn_safe_mode.bat
  → 该模式下跳过托盘初始化、窗口恢复等可能卡住的环节

第三步：查看日志
  → 启动日志: ssrvpn_startup.log（诊断脚本生成）
  → 安全模式日志: ssrvpn_safe.log（安全模式脚本生成）
  → 应用日志: %LOCALAPPDATA%\SSRVPN\logs\startup.log

═══════════════════════════════════════
  常见问题
═══════════════════════════════════════

Q: 双击后什么反应都没有？
A: 很可能是 SmartScreen 拦截或缺少渲染组件
   1. 右键 exe → 属性 → 勾选"解除锁定"
   2. 运行 SSRVPN_Diag.bat 查看完整诊断

Q: 提示缺少 DLL？
A: 运行 SSRVPN_Diag.bat 会列出具体缺失的文件
   正常发布包已自带 VC++ 运行库和 d3dcompiler_47.dll
   如果缺失，通常是 ZIP 没有完整解压或文件被安全软件隔离

Q: 杀毒软件报毒？
A: SSRVPN 和 Mihomo 核心均未签名，部分杀软会误报
   将程序目录加入杀软信任区即可

Q: 连接后无法上网？
A: 在设置里检查 TUN 模式和代理端口配置
   如使用代理模式，需手动设置系统代理为 127.0.0.1:7890
