SSRVPN Windows 便携版 v2.4.5

═══════════════════════════════════════
  从 v2.4.5 或更早版本升级
═══════════════════════════════════════

如果你曾运行过旧版的 CET 修复脚本，请在升级后右键
remove_legacy_cet_exemption.bat，选择“以管理员身份运行”一次。
它只删除旧版 SSRVPN 留下的进程安全例外，不会关闭 Windows 安全功能。

═══════════════════════════════════════
  使用方法
═══════════════════════════════════════

1. 完整解压 ZIP 到文件夹（⚠️ 不要在压缩包预览窗口中直接运行）
2. 右键 ssrvpn_windows.exe → 属性 → 如果看到"解除锁定"，勾选后点确定
   这一步很重要！否则 Windows SmartScreen 可能静默拦截程序。
3. 双击主目录里唯一的 exe：ssrvpn_windows.exe 启动
4. 粘贴你的订阅链接或节点配置
5. 打开主页的连接开关

═══════════════════════════════════════
  打不开怎么办？
═══════════════════════════════════════

第一步：双击运行 SSRVPN_Diag.bat
  → 自动诊断缺失文件、SmartScreen 拦截等问题
  → 生成 ssrvpn_diag.log、ssrvpn_startup.log、ssrvpn_startup.err.log

第二步：如果诊断未解决，尝试安全模式
  → 双击运行 ssrvpn_safe_mode.bat
  → 该模式下跳过托盘初始化、窗口恢复等可能卡住的环节

第三步：查看日志
  → 启动日志: ssrvpn_startup.log（诊断脚本生成）
  → 安全模式日志: ssrvpn_safe.log（安全模式脚本生成）
  → 应用日志: %LOCALAPPDATA%\SSRVPN\logs\startup.log

═══════════════════════════════════════
  常见问题
═══════════════════════════════════════

Q: 双击后什么反应都没有？
A: 很可能是 SmartScreen 拦截、主目录运行库缺失或缺少渲染组件
   1. 右键 exe → 属性 → 勾选"解除锁定"
   2. 运行 SSRVPN_Diag.bat 查看完整诊断

Q: 提示缺少 DLL？
A: 运行 SSRVPN_Diag.bat 会列出具体缺失的文件
   正常发布包会在主目录和 bin 自带 VC++ 运行库，并在 bin 自带 d3dcompiler_47.dll
   如果缺失，通常是 ZIP 没有完整解压或文件被安全软件隔离

Q: 杀毒软件报毒？
A: SSRVPN 和 Mihomo 核心均未签名，部分杀软会误报
   将程序目录加入杀软信任区即可

Q: 连接后无法上网？
A: 在设置里检查 TUN 模式和代理端口配置。
   使用系统代理模式时，SSRVPN 会在连接后设置 127.0.0.1:7890，
   断开连接后恢复原系统代理设置。
