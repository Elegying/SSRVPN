# ADR-004：三端只保留首页与订阅

## 状态

已接受

## 日期

2026-07-18

## 背景

SSRVPN 的核心任务是导入订阅、选择节点并建立连接。原“解锁测试”页面通过第三方网站和
接口推断流媒体或服务可用性，结果容易受页面结构、地区策略、账号状态和临时网络故障影响。
它既不能证明账号权益，也不参与 VPN 连接闭环，却额外引入共享网络探测服务、三端页面、
导航入口和一套持续变化的判定规则。

用户已明确要求三端删除该功能，把产品表面收敛到首页与订阅。历史版本中的实现和说明继续
留在 Git 历史与历史审查材料中，但不能作为当前产品能力。

## 决策

1. Android、macOS 和 Windows 的主导航只包含“首页/主页”与“订阅”两项。
2. 删除三端解锁测试页面、共享桌面 UI、共享检测服务及其专属测试，不保留隐藏入口、开关、
   路由、死代码或兼容门面。
3. 当前用户指南、排障、健康状态和产品行为要求不得继续宣传或指导使用解锁测试；历史变更
   日志和带日期的审查快照可以保留原始记录。
4. 仓库门禁同时检查删除路径、活动 Dart 源码中的功能标识、当前用户文档、三端页面组合和
   共享的两项导航定义，阻止残留或误恢复。
5. 不以其他网络探测、网页快捷入口或“实验功能”替代该页面。未来若确有用户需求，必须用
   新决策说明可验证的用户价值、隐私边界和跨平台测试成本。

## 结果

- 三端产品结构一致，用户只需理解连接与订阅两个主任务。
- 删除不可靠的外部判定和维护成本，减少网络请求、共享服务与 UI 状态分支。
- 现有订阅、节点、连接、更新和诊断行为不受影响；HTTP 订阅策略也不因本决策改变。
- 历史文档仍可解释旧版本，但当前文档和源码不会让用户误以为该能力仍然存在。

## 未采用的方案

### 保留页面但隐藏导航

隐藏入口仍会保留网络服务、依赖和回归面，也容易被深链或后续重构意外重新暴露，因此拒绝。

### 只删除 UI，保留共享服务

无人调用的检测服务属于死代码，会让维护者误判产品边界并继续承担安全与兼容成本，因此拒绝。

### 保留少量网站快捷检测

网页可访问不等于账号或地区权益可用，仍会产生无法可靠解释的结果，因此拒绝。

## 验证守卫

- `scripts/check-product-surface-guards.sh` 检查删除路径、活动源码、当前文档、共享页面、两项导航、
  主页保留“关于”但订阅页不得出现该入口，以及离线节点预选择契约。
- `make verify` 运行三端分析、测试、覆盖率、原生测试和上述产品表面守卫。
