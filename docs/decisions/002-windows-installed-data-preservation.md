# ADR-002：Windows 安装版数据保留与多来源隔离

## 状态

已接受

## 日期

2026-07-16

## 背景

早期 Windows 安装器会搜索桌面、下载目录或旧安装位置中的便携数据，并尝试备份、合并或
恢复到新安装。用户机器上存在多个旧副本、冲突文件或不完整恢复状态时，这条迁移链可能
直接阻止安装。`v3.3.3` 因此临时改为升级时删除安装版数据，以彻底绕开多来源迁移失败。

自 `v3.4.2` 起，安装器已经不再发现、选择或合并任何便携数据源。它使用固定的每用户安装
目录，只在复制前验证并停止当前安装实例，随后替换程序文件。因此，保留固定安装目录内的
用户数据不会重新引入“多个本地数据源导致安装失败”的原问题。

## 决策

1. 安装版固定使用 `%LOCALAPPDATA%\Programs\SSRVPN`，不提供安装目录选择，也不需要管理
   员权限。
2. 安装版的首选数据目录为 `{app}\bin\ssrvpn`。覆盖升级、卸载以及保留数据后的重装都
   保留该目录中的设置、订阅、DPAPI 密钥与运行状态，只替换或删除程序文件。
3. `%LOCALAPPDATA%\SSRVPN\ssrvpn` 仅作为应用无法写入可执行文件旁数据目录时的运行时
   回退目录。安装器保留但不合并该目录，也不因它与首选目录同时存在而阻止安装。
4. 安装器不得搜索、修改或迁移桌面、下载目录和其他位置的便携副本；多个便携副本可以
   共存，但不属于安装事务。
5. 旧代理恢复状态、安装确认状态以及两个已知 WebView2 缓存目录可以按精确路径清理；不得
   扩大到用户数据父目录。
6. 用户主动要求清空数据时，通过应用内重置或明确的人工删除流程完成，不把每次升级当作
   隐式重置。

## 结果

- 多个旧数据源不会再参与安装，因冲突数据、损坏备份或恢复失败而中止安装的路径被移除。
- 普通覆盖升级保留订阅和设置，不再要求用户每次重新导入。
- 机器上可能继续存在未使用的便携或回退数据；它们不会影响安装，但仍由当前用户负责清理
  和保护。
- 应用在首选数据目录不可写时仍可能执行一次受白名单和冲突检查保护的运行时回退迁移；这
  是应用启动边界，不是安装器的多来源发现流程，关键文件冲突继续安全失败并保留证据。

## 取代的旧决策

本决策取代 `v3.3.3`–`v3.4.1` 的“每次安装删除用户数据”临时策略，以及 ADR-001 中关于
Windows 安装器清理或保留数据的条款；ADR-001 的 DPAPI 与 secret 存储边界继续有效。

## 未采用的方案

### 每次升级删除全部安装版数据

能够绕开迁移缺陷，但在安装器已经隔离多来源后只会造成不必要的数据丢失，因此不再采用。

### 自动搜索并合并所有便携副本

无法可靠判断哪个副本是用户期望的唯一真源，冲突和损坏还会重新进入安装关键路径，因此
拒绝。

### 在多个数据目录之间自动择新覆盖

时间戳不能证明数据完整性或用户意图，也可能用损坏或错误账户的数据覆盖有效配置，因此
拒绝。

## 验证守卫

- `scripts/test_windows_installer_config.py` 检查安装器不含便携数据发现/迁移流程。
- 同一测试文件检查 `[InstallDelete]` 仅替换程序文件并保留两个安装版数据目录。
- `scripts/check-windows-installer-structure.sh` 继续验证固定每用户安装目录和权限边界。
