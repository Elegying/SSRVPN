# 性能基线

性能门禁关注“关键路径仍然能在有界输入上稳定执行”，不把某台机器的微秒数当作跨平台硬阈值。当前覆盖三条不依赖公网的共享路径：

- 解析单一 YAML 订阅；
- 合并多个订阅并处理名称和来源；
- 从合并结果生成 Mihomo 配置。

## 运行方式

快速结构校验已纳入 `make verify`：

```bash
make performance
```

采集可比较的本机基线：

```bash
cd packages/ssrvpn_shared
dart run tool/benchmark_critical_paths.dart
```

工具输出 JSON，包含输入节点数、来源数、输入/输出体积和各路径的最小、中位、最大耗时。`--smoke --verify` 使用较小样本并校验节点数和配置结构；任何无效结果都会以非零状态退出。

2026-07-14 的首个参考样本来自 Apple M1、macOS 26.5.2、Flutter 3.44.1：每来源 1200 个节点、4 个来源、7 次测量时，中位数分别为解析 17,358 μs、合并 92,825 μs、配置生成 174,603 μs。它只用于同环境后续比较，不是其他机器的通过阈值。

## 比较规则

只有以下条件同时接近时才比较耗时：

1. 同一操作系统、CPU 架构和电源模式；
2. 同一 Flutter/Dart 主版本；
3. 没有同时运行重负载任务；
4. 使用相同工具模式和 fixture 规模。

单次结果只用于定位，不用于宣称性能提升。连续三次基线的中位数都比同环境历史值高 25% 以上时，才视为需要调查的退化信号；先使用 profiler 找原因，再决定优化或记录合理变化。不得为了通过门禁降低输入规模、跳过输出校验或加入机器相关的宽松超时。

## 解释边界

该工具测量纯 Dart 共享逻辑，不代表真实订阅下载、磁盘、Mihomo 启动、VPN 建链或 UI 帧性能。上述路径应分别使用目标平台 profile 构建、系统性能工具和真机网络场景验证。Windows 11 真机矩阵仍按项目健康文档单独执行，不由本基线替代。
