# 路线图

路线图只保留能提高用户结果且有明确验收证据的工作。已完成内容进入 `CHANGELOG.md`，不在这里重复堆积。

统一诊断与恢复、读屏语义和键盘测试、关键路径性能基线、订阅/诊断/私有存储职责拆分、
Android 应用模块内置 Kotlin、Windows 中文无提权安装与核心单次恢复已完成并进入
`make verify`。桌面固定走免费分发；系统不会验证受信任发布者，文档和 UI 必须持续如实提示。

### 1. 真机无障碍与生命周期矩阵

- 用 TalkBack、VoiceOver 和 Narrator 走完首次导入、节点选择、连接/取消、错误重试和断开。
- Android 真机验证首次授权、后台重启、快捷磁贴、通知断开、休眠/唤醒和弱网取消。
- macOS 验证系统代理和 TUN 的授权取消、连接、异常退出、Dock 重开与网络设置恢复。

完成标准：关键流程有平台与系统版本、读屏器、步骤、结果和截图/日志证据；现有 Widget
语义测试继续作为回归门禁，但不冒充真机验收。

### 2. 按变更风险执行 Windows 发布烟雾测试

现有 Windows 11 UAT 作为当前基线，不再为不触达 Windows 的改动重复完整矩阵。后续修改
安装器、代理恢复、guardian、TUN 或内置更新，或准备新的正式版本时，再在安装版完成对应
首次启动、覆盖升级、连接/断开、数据保留和卸载烟雾测试。

完成标准：记录版本、提交、系统版本、安装包哈希、触达边界和结果；发现的问题先进入自动
回归，再修改高风险实现。

## 后续改进

### 可维护性

- 只按可独立测试的用户行为拆分大型桌面 UI 和服务文件。
- 保持共享领域逻辑单一来源，平台层只承载 OS 集成与差异文案。
- 第三方 Flutter 插件完成 AGP 9 迁移后，删除 Android 全局 `builtInKotlin=false` 与旧 KGP
  兼容声明；应用模块本身不得回退旧插件。

完成标准：每个切片减少职责或重复，并保持 `make verify` 与目标平台构建通过；不接受只有行数变化的重构。

### 可观测性与性能

- 在低端 Android 与目标桌面机重复现有订阅解析、合并、配置生成基线，再补充启动、连接
  和取消的非敏感阶段耗时。
- 诊断报告继续保持默认脱敏、有大小上限并由用户主动复制；只有明确用户需求时再增加
  文件导出，不默认收集或上传。

完成标准：基线脚本、样本条件和前后数据可复现，优化不牺牲取消与回滚语义。

## 不作为当前目标

- 不购买 Apple Developer ID 或 Windows Authenticode 证书，不恢复付费桌面签名自动化。
- 不在没有稳定 macOS 代码身份时把 `.api-secret` 迁入依赖代码身份 ACL 的 Keychain。
- 不为追求目录一致而强行共享三端 UI。
- 不在没有证据时替换 Mihomo、状态管理方案或发布系统。
- 不用排除源码、无断言测试或大量模板文档制造“评分提升”。
- 不恢复 setuid 核心，也不把 macOS TUN 提权变成持久 root 安装。

当前评分与风险依据见 [项目健康状态](PROJECT_HEALTH.md)。
