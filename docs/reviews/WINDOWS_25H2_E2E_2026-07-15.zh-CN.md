## 判定标准与基线

- `PASS`：当前实机已有完整证据。
- `FAIL`：问题已稳定复现。
- `INCONCLUSIVE`：前置条件或取证不足，必须重测。
- `BLOCKED`：缺少目标系统或待构建安装包，尚不能执行。

测试开始前确认 WinINet 系统代理关闭、WinHTTP 直连，没有 SSRVPN 核心进程、本地代理监听或待处理恢复记录。正常断开、取消连接和异常退出均以准确恢复该基线为通过条件。

## 异常核心退出时间线

第一次强制结束核心：

```text
T+0 ms     强制结束核心
T+60 ms    本地监听消失
T+561 ms   系统代理关闭
T+1027 ms  新核心进程出现
T+1134 ms  本地监听恢复
T+2316 ms  系统代理重新启用
```

监听消失到代理关闭之间约有 501 ms 的死代理窗口。自动恢复只在本地监听恢复后才重新启用系统代理。

第二次强制结束核心：

```text
T+0 ms     再次强制结束核心
T+67 ms    本地监听消失
T+547 ms   系统代理关闭
T+1527 ms  仍无核心进程，客户端保持断开
```

本轮死代理窗口约 480 ms。客户端没有无限重启核心，而是在第二次故障后进入安全断开。

## 已复现问题与源码修复

### W25H2-01：代理恢复记录删除竞态

状态：**3.4.1 FAIL / 3.4.2 源码已修复 / 待二进制实测**

launcher 已恢复代理并删除原生注册表记录，但 Dart JSON 记录仍存在时，应用重启会再次执行恢复。Windows PowerShell 5.1 在不存在的注册表路径上执行 `Remove-Item` 会返回非零状态，导致恢复永久停在 `recoveryPending`。同一缺陷也影响首次创建原生备份前的清理。

修复为在两处删除前使用 `Test-Path -LiteralPath`，只在路径存在时执行 `Remove-Item -LiteralPath`。代理与关闭恢复静态回归 12/12 通过，Windows PowerShell 5.1 兼容检查 7/7 通过。

### W25H2-02：正常退出时 Flutter 子窗口销毁重入

状态：**3.4.1 FAIL / 3.4.2 源码已修复 / 待二进制实测**

3.4.1 连续五次正常退出均在 Flutter 子 HWND 处理父窗口销毁通知时发生 `WM_PARENTNOTIFY` / `WM_DESTROY` 重入，并从空 view 对象偏移读取。修复后的顺序为：消息循环结束、显式 `window.Destroy()`、`CoUninitialize()`、标记正常关闭。

### W25H2-03：安装器提前启动客户端

状态：**3.4.1 FAIL / 3.4.2 安装脚本已修复 / 待安装包实测**

启动项改为完成页的可选“启动 SSRVPN”，并加入 `postinstall` 和 `skipifsilent`。静默安装不再隐式启动 GUI，安装包 smoke test 会在安装后显式启动 launcher。

### W25H2-04：WebView2 缓存未清理

状态：**3.4.1 FAIL / 3.4.2 安装脚本已修复 / 待安装与卸载实测**

升级安装和卸载会精确清理两个已知 `EBWebView` 子树，不删除其父目录。安装器配置回归 18/18 通过。

### W25H2-05：在线升级确定性清空订阅与设置

状态：**3.4.1 已知设计 / 3.4.2 源码已修复 / 待安装包实测**

3.3.3 起的安装器会同时删除安装目录内数据和 LocalAppData 回退数据。因此旧逻辑下的内置更新即使安装成功，也会确定性清空订阅、设置和 DPAPI 密钥。3.4.2 改为只替换安装目录根文件、`bin` 顶层程序文件和 Flutter 运行时目录，明确保留安装版数据、LocalAppData 回退数据和窗口状态；CI 会用同一安装包执行二次覆盖并验证保留哨兵。

### W25H2-06：PowerShell 更新兜底可能随旧进程一起被终止

状态：**3.4.1 风险确认 / 3.4.2 源码已修复 / 待运行中升级实测**

Flutter 应用属于外层 launcher 的 Job Object。旧 PowerShell fallback 及其启动的安装器没有可验证的 breakaway 标志；安装器关闭旧应用后，launcher 的 `TerminateJobObject` 可能把安装器一起终止。3.4.2 先用 `GetShellWindow` 确认已有桌面 Shell，再通过现有 Windows Explorer shell 交接安装包；Shell 不存在或交接失败时显示已校验文件路径，要求退出后手动运行，不再从受约束的应用进程创建兜底安装链。

### W25H2-07：代理未恢复仍停止核心

状态：**3.4.1 风险确认 / 3.4.2 源码故障注入 PASS / 待安装包实测**

旧安装脚本在精确恢复和禁用自有端点都失败后仍继续停止应用与核心，并可能返回成功。3.4.2 在任何 `Stop-Process` 前重新读取 Internet Settings：仍指向恢复记录中的端点、仍符合 SSRVPN 指纹、精确安装路径进程仍运行且系统代理指向本地端点，或注册表无法确认时，以非零状态阻断安装和卸载。当前实机已注入无恢复记录的完整指纹、支持字段被改变且精确路径核心仍运行的部分指纹，以及“SSRVPN 覆盖前原本就是另一 localhost 代理”的有效恢复记录。前两轮以退出码 3 阻断且测试核心保持存活；第三轮以退出码 0 停止核心并准确保留原 localhost 代理；三轮均逐字段恢复测试前代理基线。

### W25H2-08：Windows 规范化 AutoDetect=0 后误走部分恢复

状态：**实机观察 / 3.4.2 源码已修复 / 待二进制实测**

实机故障注入期间观察到 Windows 可把 `AutoDetect=0` 规范化为值缺失；两者语义均为关闭。旧 Dart、PowerShell 和原生恢复判断要求该值必须存在，可能只恢复代理端点并遗留支持字段。3.4.2 统一把“值缺失或 DWORD 0”视为关闭，存在但类型错误或非零仍拒绝完整所有权。
