# SSRVPN 常见问题排查

先确认安装包来自正式 Release 或官网固定下载地址，并记录平台、SSRVPN 版本、复现步骤
和实际现象。以下条目不使用内部错误码，界面文字随版本变化时以实际提示为准。

## 导入后没有节点

**现象**

导入订阅或节点后，主页仍为空，或者刷新立即失败。

**原因或检查**

- 链接复制不完整、已经失效，或服务商暂时不可达。
- 当前网络、DNS、其他 VPN 或代理阻止了订阅下载。
- 导入内容不是应用支持的订阅或节点格式。

**动作**

1. 确认使用完整的 HTTPS 链接，且前后没有多余空格。
2. 暂停其他 VPN 或代理，再到“订阅”页面刷新一次。
3. 向订阅服务商确认链接仍有效；不要把完整链接贴到公开 Issue。

## 点击连接后一直等待或很快断开

**现象**

状态长时间停在连接中，或刚显示连接便恢复未连接。

**原因或检查**

- 当前节点已经超时或订阅信息陈旧。
- TUN 所需的系统权限没有获得，或系统中已有其他 TUN/VPN 占用网络。
- 核心启动后无法通过当前节点建立可用网络。

**动作**

1. 刷新订阅并重新测试节点，选择一个未超时的节点。
2. 退出其他 VPN、代理和网络过滤工具后重试。
3. 桌面端先用系统代理验证节点；需要 TUN 时再按平台指南授予权限。
4. 仍失败时完全退出 SSRVPN 后重新打开，并保存已脱敏日志用于排查。

## 显示已连接但网页或应用无法联网

**现象**

SSRVPN 显示已连接，但网页打不开，或只有部分应用不走代理。

**原因或检查**

- 节点可连接但出口不可用，或目标网站拒绝该出口。
- 系统代理只覆盖遵循系统代理设置的应用。
- 其他代理软件在连接后又修改了系统代理或路由。

**动作**

1. 切换另一个节点并等待连接验证完成。
2. 浏览器可用而游戏或桌面客户端不可用时，按平台说明改用 TUN。
3. 关闭其他代理软件；断开 SSRVPN 后确认普通网络已经恢复，再重新连接。

## 断开后系统代理看起来不正确

**现象**

断开或退出后，浏览器仍显示代理错误，或者系统代理值与预期不同。

**原因或检查**

SSRVPN 只恢复能够证明由自己设置、且之后没有被其他软件改写的代理状态。多个代理软件
同时运行时，最后一次修改可能来自其他程序。

**动作**

1. 退出所有代理和 VPN 软件。
2. 在系统网络设置中检查代理开关，再测试直连网络。
3. 如仍异常，重新启动系统后先测试普通网络，再单独启动 SSRVPN。

## 解锁检测显示“无法判断”或“检测失败”

**现象**

网页似乎能打开，但解锁检测没有显示“支持”。

**原因或检查**

“可访问”不等于账号或地区功能已解锁。官方页面、接口或返回格式变化时，SSRVPN 会
保守返回“无法判断”；网络或服务异常则可能显示“检测失败”。

**动作**

确认连接稳定后重试一次，或切换节点比较结果。不要把“无法判断”解释为“支持”。

## 首页没有显示 IPv6

**现象**

节点或 TUN 支持 IPv6，但首页公网 IP 仍只显示 IPv4。

**原因或检查**

这是产品设计。首页固定展示 IPv4，避免把一次公网 IP 查询误当成 IPv6 连通性证明。

**动作**

无需修复。IPv6 是否真正可用取决于本地网络、节点、DNS 和目标服务。

## macOS 无法直接打开应用

**现象**

首次打开时 macOS 阻止运行，或提示无法验证开发者。

**原因或检查**

当前公开 DMG 未使用 Developer ID 公证，Gatekeeper 会显示安全提示。

**动作**

只在确认安装包来自正式下载地址并核对 SHA256 后，到“应用程序”中右键 SSRVPN，选择
“打开”。不要用关闭系统安全功能的方式绕过提示。

## macOS TUN 授权取消或启动失败

**现象**

选择 TUN 后出现系统管理员授权窗口；取消、等待过久或授权后仍未连接。

**原因或检查**

- 用户取消了系统授权，或授权流程超时。
- 其他 VPN/TUN 正在占用路由。
- 核心或临时运行文件未通过启动检查。

**动作**

退出其他 VPN，重新连接并在系统窗口中完成授权。密码由 macOS 处理，不应输入到
SSRVPN 自己的页面。仍失败时断开并退出应用，确认网络恢复后再收集脱敏日志。

## Windows 安装包出现 SmartScreen 提示

**现象**

Windows 提示未知发布者，浏览器也可能提示保留或删除下载文件。

**原因或检查**

当前 Windows 安装包和便携包没有 Authenticode 签名。

**动作**

仅在文件来自正式下载地址且 SHA256 匹配时选择保留并继续运行。来源或哈希不确定时不要
绕过系统提示。

## Windows 便携版提示找不到应用文件

**现象**

启动器无法找到 SSRVPN 应用文件。

**原因或检查**

通常是只复制了顶层启动器，或直接从压缩包运行而没有完整解压。

**动作**

重新下载 ZIP，将整个目录完整解压后再运行顶层 `ssrvpn_windows.exe`。普通用户优先使用
`SSRVPN_Setup.exe` 安装版。

## Windows 提示当前账户无法解密本机密钥

**现象**

启动页提示“当前 Windows 账户无法解密本机密钥”，并显示一个可复制的
`.api-secret.dpapi` 完整路径。应用不会自动删除或覆盖该文件。

**原因或检查**

该文件使用创建它的 Windows 登录用户通过 DPAPI 加密。把便携目录连同密钥文件复制到
另一台电脑或另一个 Windows 账户，或者密文文件损坏时，当前账户无法解密。订阅和设置
文件不依赖这次解密，仍应保留。

**动作**

1. 完全退出 SSRVPN，不要删除、上传或分享该密钥文件。
2. 按界面显示的完整路径找到 `.api-secret.dpapi`。常见位置是安装版的
   `%LOCALAPPDATA%\Programs\SSRVPN\bin\ssrvpn`、回退目录的
   `%LOCALAPPDATA%\SSRVPN\ssrvpn`，或便携目录的 `bin\ssrvpn`。
3. 在原目录把文件重命名为
   `.api-secret.dpapi.unreadable-YYYYMMDD-HHMMSS`，保留它作为恢复证据。
4. 重新启动 SSRVPN。应用会为当前 Windows 账户生成新密钥；已有设置和订阅继续保留。

只有原 Windows 账户和原设备上下文可能解开旧密文。若要跨电脑或跨账户移动便携版，
复制前应退出 SSRVPN，并从副本中排除 `.api-secret.dpapi`，让目标账户首次启动时生成新值。

## Windows 安装后提示旧数据尚未安全恢复

**现象**

安装器完成文件复制后不自动启动 SSRVPN，并提示旧数据尚未安全恢复。

**原因或检查**

安装器发现新目录中的同名设置、订阅或加密密钥与升级前的校验备份不同，因此拒绝静默
覆盖。恢复状态和备份会留在 `%LOCALAPPDATA%\SSRVPN\installer` 与
`%LOCALAPPDATA%\SSRVPN\installer-recovery`。

**动作**

1. 完全退出 SSRVPN，不要修改或删除 `installer`、`installer-recovery` 中的状态与备份。
2. 在活动目录 `%LOCALAPPDATA%\Programs\SSRVPN\bin\ssrvpn` 中，把本次安装产生且与
   旧备份冲突的同名文件另行重命名留存，例如追加
   `.postinstall-conflict-YYYYMMDD-HHMMSS`；不要用它覆盖恢复目录中的副本。可能涉及
   `.api-secret.dpapi`、`settings.json`、`subscriptions.json`、缓存、配置或 GeoIP 文件。
3. 重新运行同一版本安装器。只有逐文件 SHA256 清单完整、目标无内容冲突时，安装器才
   恢复旧数据并自动启动。
4. 启动后核对订阅和设置，再决定是否清理另行留存的冲突副本。无法判断应保留哪份时，
   先打包安装日志、恢复状态和两份文件清单，通过私密渠道求助；密钥和订阅不能公开上传。

## Windows 关闭窗口后找不到应用

**现象**

关闭主窗口后应用仍在运行，但任务栏没有窗口。

**原因或检查**

启用最小化到托盘时，关闭窗口会隐藏到系统托盘。

**动作**

从系统托盘打开 SSRVPN 菜单并选择显示窗口。若窗口仍无法出现，可完全退出后使用发布包
中的 `ssrvpn_safe_mode.bat` 启动；`SSRVPN_Diag.bat` 可检查便携包必需文件。

## Android 首次连接没有建立 VPN

**现象**

点击连接后系统没有建立 VPN，或应用很快回到未连接。

**原因或检查**

- 系统 VPN 授权未确认。
- 设备中已有另一个 VPN 正在运行。
- 系统后台限制终止了前台 VPN 服务。

**动作**

关闭其他 VPN，重新连接并确认 Android 的系统 VPN 授权。连接期间不要强制停止应用；如
系统限制后台运行，请在系统应用设置中允许 SSRVPN 正常运行。

## 更新下载或安装失败

**现象**

应用发现新版本，但下载、校验或覆盖安装失败。

**原因或检查**

- 当前网络无法访问主下载源或备用 Release。
- 下载文件不完整，SHA256 校验失败。
- Android 已安装包与正式 Release 使用的签名身份不同。

**动作**

1. 切换到稳定网络后重试，不要安装校验失败的文件。
2. 从正式 Release 手动下载对应平台产物并核对 SHA256。
3. Android 覆盖安装仍失败时，先确认现有应用是否来自正式渠道；卸载会清除本地数据，
   不要在未记录订阅和设置前直接卸载。

## 提交问题前

请提供平台、SSRVPN 版本、安装版或便携版、复现步骤、预期结果和实际结果。日志必须先
移除订阅 URL、Token、节点地址、密码和 API secret。Windows 崩溃转储可能包含敏感数据，
不要上传到公开 Issue；需要进一步分析时应通过维护者指定的私密渠道传递。
