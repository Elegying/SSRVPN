# SSRVPN 公共用户指南

本指南说明 Android、macOS 和 Windows 共用的操作。安装、系统权限和平台限制请先
阅读对应页面：

- [Android 安装与权限](../SSRVPN_Android/USER_GUIDE.md)
- [macOS 安装与权限](../SSRVPN_MacOS/USER_GUIDE.md)
- [Windows 安装、便携版与权限](../SSRVPN_Windows/USER_GUIDE.md)

## 导入订阅或节点

1. 首次启动且没有节点时，应用会提示添加订阅或节点。
2. 粘贴服务商提供的完整 HTTPS 订阅链接，或应用支持的单节点链接。
3. 确认后等待下载和解析完成，再返回主页查看节点。
4. 如果没有自动显示节点，进入“订阅”页面手动刷新；仍失败时参阅
   [常见问题排查](TROUBLESHOOTING.zh-CN.md)。

订阅链接通常包含可直接访问账号数据的 Token，应像密码一样保管。不要把完整链接
放进截图、Issue、聊天记录或公开日志，也不要使用来源不明的订阅。

## 连接、切换与断开

1. 在主页选择一个未超时的节点。超时节点会变灰并移到列表后方，仍可进入编辑页检查。
2. 桌面端按需要选择系统代理或 TUN；Android 首次连接时确认系统 VPN 授权。
3. 点击主页连接按钮并等待状态变为“已连接”。
4. 连接期间可以选择其他可用节点直接切换，不需要先断开。
5. 不再使用时点击断开。桌面端若要完全退出，还应从菜单栏或系统托盘选择“退出”。

系统代理主要覆盖遵循系统代理设置的应用；游戏、部分桌面客户端及其他不读取系统代理
的程序通常需要 TUN。TUN 涉及更高系统权限，具体要求以对应平台页面为准。

## 正确理解状态

- “连接中”只表示正在启动核心、申请权限或验证网络，不等于已经可用。
- “已连接”表示本地连接链路已经建立；节点出口仍可能临时故障。网页打不开时先切换
  节点，不要仅凭按钮颜色判断网络一定正常。
- 延迟数值只反映一次探测结果，不证明所有网站、应用或地区服务都可访问。
- 首页公网 IP 固定只显示 IPv4；没有显示 IPv6 不代表 IPv6 节点或 TUN 流量不受支持。
- 解锁检测采用保守判断。“支持”需要服务专用证据；“可访问”不等于账号或地区功能
  已解锁；证据不足会显示“无法判断”，网络或服务异常可能显示“检测失败”。

## 更新订阅

1. 打开“订阅”页面。
2. 对目标订阅执行刷新，或使用页面提供的全部刷新操作。
3. 刷新完成后回到主页，确认节点数量、名称和当前选择符合预期。
4. 如果正在连接，等待刷新完成后再快速连续切换或重复连接。

订阅节点以服务端返回内容为准，刷新可能覆盖该订阅内节点的本地字段。重要的手工调整
应先记录；不要通过反复删除和重新添加来掩盖网络或订阅本身的问题。

## 隐私与安全提示

- 只使用可信服务商提供的 HTTPS 订阅。HTTP 会让订阅 Token 和节点信息面临被窃听或
  篡改的风险。
- 仅从项目正式 Release 或官网固定下载地址获取安装包，并按发布页提供的 SHA256
  校验文件核对下载内容。
- 不要公开订阅 URL、节点密码、API secret、配置目录、崩溃转储或未经检查的完整日志。
- macOS 把长期 API secret 从普通设置中分离到权限为 `0600` 的专用文件，Windows 使用
  与当前登录用户绑定的 DPAPI 加密存储。Mihomo 运行期间的 `config.yaml` 仍需要包含
  该 secret，因此不要共享运行时配置或整个数据目录。
- Windows 便携目录会保存订阅、缓存和运行时配置。不要把正在使用的数据目录放进公共
  网盘、共享账号目录或交给其他人。
- macOS 管理员密码由系统授权窗口处理，SSRVPN 不读取或保存该密码。
- 同一系统账号下的恶意进程仍可能读取当前用户可访问的运行时文件；凭据隔离或系统安全
  存储不能替代受保护的系统账号、磁盘加密和可信下载来源。

## 遇到问题

先断开其他 VPN 或代理软件，记录平台、SSRVPN 版本、操作步骤和实际现象，再查看
[常见问题排查](TROUBLESHOOTING.zh-CN.md)。提交日志前必须移除订阅链接、Token、
节点地址和密码；不要在公开 Issue 中上传崩溃转储。
