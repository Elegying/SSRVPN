# 项目健康状态

最近审查：2026-07-14<br>
审查基线：基于 `v3.3.5` 之后的 `main` 修复与可靠性重构，包含 Windows 无提权全新覆盖安装、主进程退出后原生代理兜底、PowerShell 5.1 假绿防护、发布事务、固定规则源、部分订阅结果、Android 单次恢复、统一诊断与脱敏报告、无障碍语义、性能基线、服务职责拆分、Android 应用模块内置 Kotlin，以及默认关闭的桌面签名自动化；这些变更尚未打标签或发布新版本。

## 结论

SSRVPN 已具备单一 Monorepo、三端发布链路、共享业务边界、可操作诊断、失败回滚和完整本地门禁。当前主要短板不再是“基本功能缺失”，而是付费证书实际启用、Windows 11 真机验收、三平台读屏器真机证据与少数平台安全边界的继续收紧。

本文件使用固定 100 分量表，替代历史审查中互不兼容的 6.8/10、7.4/10、8.5/10 等结论。分数只代表当前仓库证据，不代表应用商店审核、第三方渗透测试或所有真实网络环境。

## 继续提升到 94 分的工作

1. 在干净 Windows 11 虚拟机完成安装版与便携版的首次启动、连接、断开、退出、全新覆盖升级、损坏目录覆盖和卸载矩阵，并保存结果。
2. 配置真实 Developer ID/notarization 与 Authenticode secrets，保留签名、公证、时间戳和干净设备系统信任证据；获得稳定 macOS 代码身份后，再按验证迁移规则评估 Keychain。
3. 用 TalkBack、VoiceOver 和 Windows Narrator 验证连接主流程、订阅导入、诊断恢复和动态状态播报；现有 Widget 语义测试不能代替平台读屏器。
4. 在低端 Android、大订阅和桌面冷启动设备上重复当前性能工作负载，只有同机数据证明瓶颈后才做优化。
5. 等第三方 Flutter 插件支持 AGP 9 内置 Kotlin 后删除全局兼容开关；后续只在独立行为测试保护下继续切分核心生命周期或复杂视觉组件。
