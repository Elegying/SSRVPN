# 项目健康状态

最近审查：2026-07-17<br>
审查对象：`v3.4.5` 源码及 Windows 安装版修复；是否已经进入正式包必须以对应 tag、
Release 产物和发布后校验为准。

## 结论

SSRVPN 的源码、三端构建链和恢复模型整体健康，本轮深度复核发现的四个可直接修复问题已
关闭：Windows TUN 清理误认其他 VPN 网卡/路由、桌面 OSS 更新清单缺少独立摘要背书、本地与线上
发布工具测试清单漂移，以及非 Windows 主机运行 Windows 专属测试产生伪失败。

2026-07-17 Windows 实机报告中的安装前置误判、连接中覆盖升级死代理、手动断开误恢复、
guardian 意外退出后状态失真和卸载注册表残留均已有源码修复与回归；最终
`v3.4.5` 安装包仍需在同一 Windows 矩阵重跑以形成新的 L4 证据。

Windows 安装数据保留策略也已重新确认：安装器固定使用每用户目录，只替换程序文件，不再
搜索或合并桌面、下载目录及其他便携副本。因此多个旧数据源不会再进入安装事务，保留安装版
订阅和设置不会重新引入旧的多来源安装失败。该边界由 [ADR-002](decisions/002-windows-installed-data-preservation.md)
固定。

Windows 发布通道已进一步收敛为只生成 `SSRVPN_Setup.exe`：便携 ZIP 不再构建、更新、
发布或回滚，正式发布会事务性删除 OSS 固定便携别名；若 RAM 拒绝删除则覆盖为不可执行的
退役标记，历史不可变资产保留作审计。
该边界由 [ADR-003](decisions/003-windows-installer-only-distribution.md) 固定。

项目仍不能宣称“Windows 全面验收完成”。最终正式安装包的管理员 UAC/TUN、Windows 11 Home
25H2、真实重启/`TurnOff`、内置更新和 Narrator 矩阵仍缺 L4 证据。Apple/Windows 付费签名不
属于项目目标，不作为未完成工作；免费分发缺少受信任发布者身份仍是必须对用户说明的限制。
