# SSRVPN 产品行为要求

这份清单用于开发、测试和发布检查。改动打包、首页、订阅、节点列表、测速或连接流程时，必须确认以下行为仍然成立。

## 安装包

1. macOS 发布产物必须是可拖拽安装的 DMG：根目录包含 `SSRVPN.app` 和指向 `/Applications` 的 `Applications` 快捷方式。
2. Windows 必须同时发布每用户安装版 `SSRVPN_Setup.exe` 和解压即用的便携式 ZIP。安装器升级时应自动关闭 SSRVPN 专属进程；便携包解压后的主目录只能有一个面向用户的 `.exe`，即 `ssrvpn_windows.exe`。

## 首次使用和订阅

3. 首次打开或检测到没有节点时，自动弹窗要求用户导入订阅链接或节点链接。
4. 用户导入订阅链接或节点链接后，应用自动刷新订阅，主页自动加载节点并开始测速。

## 节点列表

5. 节点测速超时后自动置底、变灰、不可选择，但仍保留编辑入口。
6. 未超时节点保持订阅原始顺序，不按延迟重新排序。
7. 应用记忆上次连接的节点。Android 快捷磁贴一键连接时默认沿用上次节点。
